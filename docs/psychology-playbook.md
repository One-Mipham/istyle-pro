## 触点 1: App Store 列表页（驱动下载）

### 应用的模型

**🟢 Loss Aversion（损失厌恶）** — 最强杠杆
- 标题角度："不要让你的形象限制你的机会" ＞ "改变你的形象"
- 副标题："看到最好的自己——在买任何衣服之前" 暗示不用的损失

**🟢 Present Bias（当下偏好）**
- 关键词"30 秒生成""即刻预览"强调即时满足
- 截图中展示 Before/After 对比滑块——用户一眼就能看到价值和惊喜感

**🟢 Social Proof（社会证明）**
- 上线后尽快累积："100,000+ 形象已生成"
- App Store 评分目标 4.5+

### 建议

```
标题: iStyle Pro — AI 个人形象设计师
副标题: 拍照 → 30秒看效果。发型、穿搭、鞋帽 一站预览
截图第1张: Before/After 对比（最震撼的案例）
截图第2张: 风格分类浏览页
截图第3张: AI 生成效果图
```

## 触点 2: 引导流程（安装 → 首次生成）

### 应用的模型

**🟢 IKEA Effect（宜家效应）** — 这里的应用至关重要
- 用户填写档案（性别、年龄、身高、体重、偏好风格）是"投入劳动"
- 填写越多，对后续生成结果的重视程度越高
- 但需平衡：3 屏引导是上限

**🟢 Foot-in-the-Door（登门槛）** — 权限申请的顺序
- 第1屏：品牌介绍 + 价值主张（建立动机）（无请求）
- 第2屏：填写档案（小承诺，用户有动力）
- 第3屏：相机权限（大请求，但此刻用户已投入，拒绝率低）

**🟢 Default Effect（默认效应）**
- 风格偏好预选"休闲"（最大公约数），用户可取消或加选

**🟢 Activation Energy（激活能量）**
- 允许"跳过"档案填写直接拍照——降低首次使用门槛
- 但跳过时提示："填写档案后，AI 推荐准确度提升 60%"

### 建议

| 屏幕 | 内容 | 心理机制 |
|------|------|---------|
| 屏1 | "你的私人 AI 形象顾问" + 3张惊艳效果图 | Peak-End（第一印象即峰值） |
| 屏2 | 性别/年龄/身高/体重/偏好风格 | IKEA Effect + Default Effect |
| 屏3 | 相机权限申请 | Foot-in-the-Door（前2屏已投入） |
| 可选 | "跳过，直接开始" | Activation Energy 降低 |

## 触点 3: 双入口首页（行为选择架构）

### 应用的模型

**🟢 Paradox of Choice（选择悖论）** — 2个入口 = 完美
- 双入口而非多入口：快速试穿 vs 风格探索
- 这是"可管理的 2 选 1"，不会引发决策瘫痪
- **绝不添加第三个主入口**（Hick's Law 警告）

**🟢 Nudge Theory（助推）** — 引导而不限制
- "快速试穿"用更大的按钮/主色 → 推荐首次用户路径
- "风格探索"用次要按钮 → 给有明确偏好的用户

**🟢 Present Bias（当下偏好）**
- 快速试穿入口文案："拍照 → 立即看效果"（强调即时性）
- 风格探索入口文案："浏览 50+ 风格模板"（强调丰富性）

### 建议

```
┌────────────────────────────┐
│   今日剩余: 3 次免费生成    │  ← Scarcity 提示
│                            │
│  ┌──────────────────────┐  │
│  │  📸  快速试穿         │  │  ← 主按钮（60% 用户选）
│  │  拍照 → AI 推荐搭配   │  │
│  └──────────────────────┘  │
│                            │
│  ┌──────────────────────┐  │
│  │  🎨  风格探索         │  │  ← 次按钮（40% 用户选）
│  │  浏览模板 → 选择生成  │  │
│  └──────────────────────┘  │
└────────────────────────────┘
```

## 触点 4: AI 生成等待（30-60秒）

### 应用的模型

**🟢 Goal-Gradient Effect（目标梯度效应）** — 进度可视化
- 用户越接近完成，动力越强
- 步骤展示："分析体型 → 匹配风格 → 生成效果" 分步推进

**🟢 Peak-End Rule（峰终定律）** — 等待是"终"的前奏
- 等待过程本身不痛苦，但需要填充——品牌动画代替空白
- 等待是期望累积期，期待越高，揭晓时峰值越高

**🟢 Zeigarnik Effect（蔡格尼克效应）**
- "生成中…"是一个打开的循环，让用户保持注意力
- 不要隐藏进度——让它可见，创造"马上就好"的张力

### 建议

```
进度展示（非百分比，而是阶段）：

[正在分析你的体型...] ✓
[匹配最适合的风格...] → 旋转动画
[生成你的新形象...]   等待中

底部轮播风格小知识/穿搭 tips："西装起源：19世纪英国绅士…"
→ 利用等待时间传递品牌价值，而非让用户焦虑地看转圈
```

## 触点 5: Before/After 揭晓（峰值时刻）

### 应用的模型

**🟢 Peak-End Rule（峰终定律）** — 这是整个体验的"峰"
- 揭晓是用户记忆中最强烈的时刻
- 必须设计一个"哇"的瞬间

**🟢 Contrast Effect（对比效应）** — Before/After 是天生对比
- 滑块对比（用户左右拖拽看原图 vs 生成图）比两张图并排更强
- 运动是注意力磁铁——可拖拽的交互让用户停留更久

**🟢 Endowment Effect（禀赋效应）** — "这是我自己"
- 强调词："你的新形象""你的专属造型"
- 用户对自己的形象照片和生成结果有天然的拥有感

**🟢 IKEA Effect 的兑现**
- "基于你的体型和偏好"——提醒用户他们的投入产生了这个结果
- 让用户感到"我参与创造了我自己的形象"

### 建议

```
揭晓页面设计：

┌────────────────────────────┐
│  ← 拖动滑块对比 →          │
│  [Before | After]          │  ← 交互式滑块
│                            │
│  "你的休闲风格"             │
│                             │
│  💾 保存到相册              │
│  🔄 换一种风格试试          │  ← 循环回到生成
│  📤 分享（Phase 2）        │
│                            │
│  喜欢这个风格？              │
│  [查看更多休闲搭配]          │  ← 连续使用的钩子
└────────────────────────────┘
```

## 触点 6: Freemium → 付费转化

### 应用的模型

**🟢 Zero-Price Effect（零价格效应）** — Free ≠ $0
- "免费"有独特的心理吸引力
- 每日 3 次免费——足以让用户上瘾，不足以满足需求
- 免费生成的质量必须和付费完全相同（否则信任破裂）

**🟢 Endowment Effect × Loss Aversion 组合拳**

这是最强的转化机制：
> 用户连续 3 天使用免费额度 → 积累 9 张属于自己的形象照 → 第 4 天，免费额度用完后，提示："你的形象库已有 9 个造型。订阅 Pro，继续打造你的专属风格。"

用户已经"拥有"这些形象——失去继续打造的能力感到痛苦。

**🟢 Decoy Effect（诱饵效应）** — 定价设计
```
方案 A: 免费 — 3次/天
方案 B: Pro 月付 — ¥68/月 (无限生成 + 高清导出)  ← 目标
方案 C: Pro 年付 — ¥588/年 (相当于 ¥49/月, 省 ¥228)
```

方案 C 的存在让方案 B 看起来"合理"，同时为价格敏感用户提供切换选项。

**🟢 Anchoring（锚定效应）**
- 在定价页先展示年付 ¥588，再展示月付 ¥68 → ¥68 感觉便宜
- 在付费墙先展示"每次生成成本 ¥2.99",再展示"Pro 无限 ¥68/月" → ¥68 感觉超值（如果每天用 3 次,每次成本 ¥0.75）

**🟢 Rule of 100**
- Pro 月付 ¥68 是 <¥100 区间 → 用百分比打折："年付省 28%"
- 如果后续有 ¥200+ 的套餐 → 用绝对金额："年付省 ¥228"

### 建议

```
付费墙触发时机（最佳）：
- 第3天用完免费额度后（用户已有使用习惯）
- 第1次使用后立即弹出（太早，用户还没建立价值认知）❌
- 从不等用户用完（用户还没感受到损失）❌
- 免费额度用完那一次弹出 ✅

付费墙文案角度：
"今天3次已用完。
保持你的风格进化 —— ¥68/月无限生成"

而非：
"升级到 Pro"（太弱，没有动机）
```

## 触点 7: 复访与留存

### 应用的模型

**🟢 Zeigarnik Effect（蔡格尼克效应）** — 未完成循环
- Push："还有 2 次免费生成今天到期"
- "你的档案只填了 60%——完善后推荐更准"
- "3 套新风格刚上线，看看哪套适合你？"

**🟢 Reciprocity（互惠）** — 给予价值唤起回报
- 每周推送："本周为你精选了 3 套风格"
- AI 主动推荐："根据你的体型，这 5 套搭配最显瘦"
- 用户在获得免费价值后，更愿意回馈（订阅、分享）

**🟢 Mere Exposure Effect（曝光效应）**
- 品牌在通知栏的持续出现建立熟悉感
- 但频率需控制——每周 2-3 次，不超过 5 次

**🟢 Fogg Behavior Model（福格行为模型）**
- B = MAT: Behavior = Motivation × Ability × Prompt
- Motivation: 想变好看（天生强动机）
- Ability: 一键打开 App → 一键生成（降低阻力）
- Prompt: 智能通知（时机——周末早上？出门前？）

### 建议

```
通知策略：

周一 AM: "新的一周，换个新形象？" (时机：出门前)
周三 PM: "3 套'投行精英'风格上新，看看？" (新鲜感)
周五 PM: "周末约会穿什么？AI 帮你搭配" (场景触发)
周日 PM: "你的本周形象报告已生成" (个性化回顾)

每种通知角度不同，避免重复疲劳
```

## 触点 8: 社交传播（Phase 2）

### 应用的模型

**🟢 Mimetic Desire（模仿欲望）**
- 用户分享自己的形象效果图 → 朋友看到 → "我也想要"
- 这不是广告，是社交信号——"我的形象被 AI 重新设计了"

**🟢 Social Proof + Bandwagon Effect（从众效应）**
- "你的 3 位好友已在使用 iStyle Pro"（如果接入通讯录）
- "今日全球 50,000 人正在试穿新形象"

**🟢 Commitment & Consistency（承诺一致性）**
- 用户分享后，公开承诺了自己的形象意识
- 公开承诺 → 更可能持续使用和订阅

**🟢 Unity Principle（统一性原则）**
- 创作品牌社区："iStyle 用户风格挑战赛"（#我的iStyle形象）
- "我们这群注重形象的人"的身份认同

### 建议

```
分享卡片设计：
┌──────────────────────┐
│ [Before/After 对比]  │
│                      │
│ "我在 iStyle Pro      │
│  发现了全新的自己"    │
│                      │
│ 免费试你的专属造型 →  │  ← 带邀请码或链接
│ iStyle Pro           │
└──────────────────────┘

水印策略：
- 非订阅用户：右下角小水印 "iStyle Pro"
- Pro 订阅用户：无水印高清导出（订阅特权）
→ 水印成为免费推广 + 付费去水印是转化点
```

## 总结：最重要的 5 个心理杠杆（按优先级）

| # | 杠杆 | 应用点 | 影响 |
|---|------|--------|------|
| 1 | **IKEA Effect + Endowment** | 引导流程 + 揭晓时刻 | 用户因"投入了自己"而无法放弃 |
| 2 | **Contrast Effect (Before/After)** | 揭晓时刻 | 整个产品的核心"哇"时刻 |
| 3 | **Loss Aversion × Free → Paid** | 付费转化 | "不要失去你的风格进化" |
| 4 | **Peak-End Rule** | 揭晓 + 结果保存 | 用户记住峰值体验，决定是否回来 |
| 5 | **Zeigarnik Effect** | 通知/复访 | 未完成循环拉回用户 |
